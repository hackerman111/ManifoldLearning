# Источник реальных данных D01–D04 для single-index benchmark

## Цель

Эксперименты семейства D используют локальный пакет `adp_D1_data` как основной
и стандартный источник данных. Загрузка не должна зависеть от OpenML, создавать
копии больших CSV в каталоге результатов или молча принимать повреждённый файл.

## Структура источника

Корнем данных считается каталог `adp_D1_data`. По умолчанию CLI передаёт этот
путь в `SingleIndexSeriesConfig.data_dir`, но пользователь может явно указать
другой каталог с той же структурой.

Loader читает `dataset_manifest.csv` и для D01–D04 находит файл в подкаталоге
`prepared/` по полю `file`. Manifest является единственным местом соответствия
между ID сценария, именем файла, целевой переменной, ожидаемой размерностью,
SHA-256 и источником. Жёсткое дублирование этих значений в Python не требуется.

## Поток данных

1. Runner передаёт `scenario_id` и `data_dir` в loader.
2. Loader проверяет наличие manifest и единственность строки требуемого ID.
3. Путь из manifest разрешается только внутри `prepared/`; абсолютные пути и
   выход через `..` отклоняются.
4. До разбора CSV вычисляется SHA-256 и сравнивается со значением manifest.
5. CSV читается, target выбирается по manifest, остальные столбцы становятся
   признаками в исходном порядке.
6. Проверяются ожидаемые `rows`, `features`, числовой тип и конечность всех
   значений.
7. `RealDataset` возвращает массивы, фактический путь, checksum и официальный
   URL источника из manifest.

Датасет не копируется в каталог серии. В первичных CSV результатов сохраняются
только относительный путь, checksum, источник, размер и параметры split/fold.
Preprocessing и standardization обучаются отдельно внутри training fold.

## Ошибки и сетевой доступ

Отсутствующие manifest или prepared-файл дают `DatasetUnavailable`. Нарушение
схемы, небезопасный путь, несовпадение checksum, размерности либо нечисловые и
нефинитные значения дают явный `ValueError` до запуска метода.

Основной D-путь не скачивает данные. Флаг `allow_download` сохраняется в API для
обратной совместимости, но не подменяет отсутствующий или повреждённый пакет
непроверенной сетевой версией.

## Границы репозитория

Каталог `adp_D1_data` и архив `adp_D1_data.zip` считаются пользовательскими
данными. Реализация читает их на месте и не добавляет в git без отдельного
указания. В коммиты входят только код, тесты и документация.

## Проверка

TDD-проверки покрывают:

- успешную загрузку каждого prepared-файла через manifest;
- выбор правильного target и ожидаемые `(n, d)`;
- несовпадение SHA-256;
- отсутствие manifest или файла;
- небезопасный путь и повторяющийся ID;
- сохранение существующего поведения synthetic-сценариев;
- отсутствие сетевого вызова при `allow_download=True`.

После focused-тестов выполняются тесты executors, runner и storage, затем полный
`python -m pytest -q` на финальном этапе плана.
